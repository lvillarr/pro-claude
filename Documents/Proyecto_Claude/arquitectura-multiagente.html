Built-in Tools de Claude Code
Herramientas nativas disponibles para todos los agentes · configuradas en CLAUDE.md
⬛ bash
Ejecuta comandos shell: scripts Python, cron, pipes. Acceso total al sistema de archivos y procesos.
todos los agentes
📄 read_file
Lee archivos del proyecto: CSV, Excel, Markdown, JSON, código. Soporta rangos de línea para archivos grandes.
todos los agentes
✏️ write_file
Crea o sobreescribe archivos. Genera dashboards HTML, scripts Python, informes Markdown y plantillas.
todos los agentes
🔧 str_replace
Edita bloques específicos de archivos existentes. Ideal para actualizar KPIs en plantillas o ajustar scripts.
IA TD EO
🌐 web_fetch
Consulta APIs REST externas: SGL, ERP SAP, telemetría GPS, endpoints de datos en tiempo real.
TD IA
🗂️ list_dir
Lista contenido de directorios. El orquestador lo usa para descubrir archivos disponibles antes de delegar.
orquestador + todos
🔍 grep_search
Búsqueda en código y archivos de texto. Útil para encontrar patrones en logs operacionales o scripts.
IA TD
🤖 task (subagent)
Lanza subagentes en paralelo o en serie. El orquestador delega tareas atómicas a IA, TD y EO simultáneamente.
solo orquestador
Custom Skills — CLAUDE.md por agente

Procesa CSV/Excel de operaciones forestales. Estadísticas, correlaciones, análisis ON/OFF de maquinaria (SKG, FEL, CLA).
IA
ml-modeling
Modelos predictivos con scikit-learn: clasificación de fallas, series temporales de disponibilidad, clustering de comportamientos.
IA
dashboard-html
Genera dashboards HTML autocontenidos con Chart.js. Paleta Arauco, filtros, tablas de ranking con semáforo visual.
IA
llm-integration
Integra Claude API en pipelines internos. Análisis automático de reportes, prompt engineering, evaluación de modelos.
IA
api-integration
Diseño y consumo de APIs REST. Conecta SGL, ERP SAP, GPS forestal y CMMS. Documenta con OpenAPI 3.0.
TD
etl-pipeline
Construye pipelines ETL entre fuentes heterogéneas. Transforma datos de campo para consumo en SGL y Power BI.
TD
automation
Scripts de automatización con schedule/cron. Genera reportes periódicos, sincroniza datos, notifica por correo/Teams.
TD
telemetry
Captura datos de maquinaria forestal (CAN bus, GPS, sensores). Protocolos MQTT/REST. Alertas por umbral operacional.
TD
lean-management
Aplica herramientas Lean: 5S, Kaizen, A3, SMED, Poka-Yoke en contexto forestal. Gestiona rutinas SGL y cascada de objetivos.
EO
kpi-design
Diseña fichas de indicador completas: fórmula, fuente, meta, umbral, responsable. Calcula OEE forestal y brechas.
EO
process-mapping
Levanta procesos AS-IS y diseña TO-BE. Genera Value Stream Maps, identifica pérdidas y cuantifica impacto (horas, m³, $).
EO
risk-iso31000
Construye matrices de riesgo operacional bajo ISO 31000 / COSO ERM. Planes de mitigación y controles por proceso.
EO
arauco-context
Glosario corporativo, convenciones de archivos, sistemas de referencia (SGL, ERP), terminología forestal. Compartido por todos.
todos los agentes

MCP Servers — Model Context Protocol
Servidores configurados en claude_desktop_config.json o .claude/settings.json del proyecto
📁 filesystem
Acceso seguro al directorio mejora-continua/. Lee, lista y escribe archivos dentro del scope definido. Reemplaza manejo manual de paths.
todos los agentes
npx @modelcontextprotocol/server-filesystem
🗃️ sqlite
Base de datos local para históricos de KPIs, registros ON/OFF procesados y caché de resultados analíticos. Sin dependencia de red.
IA EO
npx @modelcontextprotocol/server-sqlite
🐘 postgres
Conexión a BD corporativa Arauco. Consulta tablas de operaciones, mantenimiento y costos directamente desde el agente TD.
TD
npx @modelcontextprotocol/server-postgres
📦 git
Versionamiento de scripts, CLAUDE.md y entregables. Commit, diff, log y branch desde el agente TD sin salir del contexto.
TD
npx @modelcontextprotocol/server-git
📊 excel-mcp
Lee y escribe archivos .xlsx nativamente sin convertir a CSV. Preserva formatos, fórmulas y múltiples hojas. Clave para EO.
EO IA
npx @modelcontextprotocol/server-excel
🌐 fetch
Consulta APIs externas: SGL REST API, ERP SAP RFC/SOAP, GPS forestal, CMMS. Autenticación configurable por header.
TD IA
npx @modelcontextprotocol/server-fetch
Configuración — .claude/settings.json del proyecto
// .claude/settings.json (raíz del proyecto) { "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "./datos", "./agentes"] }, "sqlite": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sqlite", "./datos/arauco_mc.db"] }, "git": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-git", "--repository", "."] }, "fetch": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-fetch"] } } }
Flujo: Informe semanal operacional
Ejemplo end-to-end · orquestador delega en paralelo a IA y EO, luego consolida con TD
1
Usuario → Orquestador
El usuario lanza: claude "genera el informe semanal operacional semana 18" desde /orquestador
arauco-context list_dir read_file
2
Orquestador analiza insumos
Lista datos/ para descubrir archivos disponibles. Identifica qué agentes necesita y si hay dependencias entre tareas.
list_dir read_file task
3
Agente IA — Análisis ON/OFF
Procesa registros de maquinaria. Calcula disponibilidad, utilización, patrones de parada y genera gráficos Chart.js.
data-analysis dashboard-html bash sqlite filesystem
4
Agente EO — KPIs y pérdidas
Extrae KPIs del período, calcula OEE, genera pareto de pérdidas y compara vs. semana anterior y meta SGL.
kpi-design lean-management python-exec excel-mcp sqlite
5
Agente TD — Sincronización SGL
Verifica que los datos del informe estén sincronizados con SGL. Si hay desvíos, los registra y genera alerta.
api-integration web_fetch fetch git
6
Orquestador consolida
Integra outputs de IA + EO + TD. Genera informe ejecutivo .docx con portada, KPIs semáforo y próximos pasos.
docx-report arauco-context write_file filesystem
7
Entregable final
Se genera en datos/ el archivo 2025-XX-XX_informe-semanal-s18.docx + dashboard HTML + commit en Git.
datos/*.docx datos/*.html git commit
